## 1. 게임 개요

### 1.1 컨셉
- **장르**: 무한 머지(합성) 퍼즐
- **플레이 방식**: 6x6 보드에서 같은 유닛을 합성하여 점수를 쌓는 엔드리스 모드
- **종료 조건**: 보드가 가득 차고 인접한 같은 유닛이 없으면 게임 오버
- **핵심 재미**: 고티어 합성의 성취감, 콤보 연쇄, 최고점수 갱신

### 1.2 합성 체인 (10단계)
```
Tier 0: 오브 (🔮)    → Tier 1: 물방울 (💧)  → Tier 2: 불꽃 (🔥)
Tier 3: 새싹 (🌿)    → Tier 4: 보석 (💎)    → Tier 5: 별 (⭐)
Tier 6: 달 (🌙)      → Tier 7: 태양 (☀️)    → Tier 8: 왕관 (👑)
Tier 9: 삼지창 (🔱)    ← 최종 단계
```

### 1.3 조작법
| 동작 | 방법 |
|------|------|
| 유닛 선택 | 유닛이 있는 셀 탭/클릭 |
| 합성 | 선택 후 인접한 같은 유닛 탭 |
| 이동 | 선택 후 인접한 빈 셀 탭 |
| 선택 해제 | 같은 셀 다시 탭 |
| 다른 유닛 선택 | 비인접 유닛 탭 |

### 4.2 게임 흐름

```
[게임 시작 startGame()]
    │
    ├── 상태 초기화 (score=0, combo=0 등)
    ├── initBoard() → 보드에 유닛 INIT_UNIT_COUNT개 배치 (tier 0~1)
    ├── renderBoard() → DOM 생성
    ├── renderTierBar() → 티어 진행 바
    ├── updateHUD()
    └── showScreen('game-screen')
          │
          ▼
    [유저 입력 루프 onCell(r, c)]
          │
          ├── 선택 없음 + 유닛 있음 → selected = {r,c}
          │
          ├── 이미 선택됨
          │     ├── 같은 셀 → 선택 해제
          │     ├── 비인접 → 재선택 또는 해제
          │     ├── 인접 + 같은 티어 → doMerge()
          │     ├── 인접 + 빈 셀 → doMove()
          │     └── 인접 + 다른 티어 → 재선택
          │
          ▼
    [합성 doMerge(fr,fc,tr,tc)]
          │
          ├── board[fr][fc] = null
          ├── board[tr][tc] = { tier: oldTier + 1 }
          ├── 점수 계산 & combo++
          ├── 이펙트 (파티클, 점수 플라이, 콤보 팝업)
          ├── NEW BEST 체크
          ├── 320ms 후: renderBoard() (합성 시 스폰 없음)
          └── checkGameOver()
                │
                ├── 빈 셀 있음 → 계속
                └── 빈 셀 없음 + 인접 같은 티어 없음
                      │
                      ▼
                [게임 오버]
                      │
                      ├── save.best 업데이트
                      ├── 700ms 후 gameover-screen 표시
                      └── 다시시작 또는 홈으로
```

### 4.3 점수 계산 공식

```javascript
// 합성 시 점수 계산 (상수는 GAME BALANCE 섹션 참조)
const tierBonus = (newTier + 1)^2 * SCORE_TIER_MULT;    // 제곱 비례 → 고티어 고득점
const comboMult = combo >= 2 ? 1 + (combo - 1) * COMBO_MULT_RATE : 1;
const pts = Math.floor((SCORE_BASE + tierBonus) * comboMult);
```

### 4.6 콤보 시스템

- **합성(merge)** 시 `combo++`
- **이동(move)** 시 `combo = 0` (리셋)
- 콤보 2 이상부터 점수에 배수 적용: `1 + (combo-1) * COMBO_MULT_RATE`
- 콤보 2 이상부터 화면에 콤보 팝업 표시
- 콤보 3 이상부터 추가 효과음 + 진동

### 4.7 특수 아이템 시스템

3종의 특수 아이템이 전용 슬롯에 저장됩니다. 게임 시작 시 0개로 초기화. **각 아이템은 최대 3개까지 보관 가능** (`ITEM_MAX = 3`). 보유 수가 3개인 상태에서 획득 조건을 달성해도 추가 지급되지 않으며, 사용하여 여유가 생기면 다시 획득할 수 있습니다.

| # | 이름 | 아이콘 | 획득 조건 | 효과 |
|---|------|--------|----------|------|
| 1 | 정화 | ✨ | 왕관(tier PURIFY_TIER) 합성 성공 | 선택한 유닛과 같은 티어 전체 삭제 |
| 2 | 폭탄 | 💣 | BOMB_COMBO_INTERVAL콤보 달성 시 마다 | 선택한 칸 중심 3x3 범위 유닛 삭제 |
| 3 | 변환 | 🔄 | 누적 MILESTONE_SCORE점 돌파 (매 마일스톤마다) | 선택한 유닛 1단계 승급 |

**사용 흐름**:
1. 아이템 슬롯 클릭 → 아이템 모드 진입 (보드에 대상 하이라이트)
2. 대상 셀 클릭 → 효과 발동, 아이템 1개 소모
3. 슬롯 재클릭 또는 빈 셀 클릭 → 모드 취소

**상태 변수**:
- `items = { purify, bomb, transform }` — 각 아이템 보유 수 (0~ITEM_MAX)
- `itemMode` — 현재 활성화된 아이템 모드 (null이면 일반 모드)
- `scoreMilestone` — MILESTONE_SCORE 단위 마일스톤 추적
- 상수 정의는 **게임 밸런스 상수** 섹션 참조

## 5. 입력 처리 상세

### 5.1 셀 클릭 흐름 (onCell)
```
onCell(r, c)
│
├── gameActive=false 또는 animating=true → 무시
│
├── selected === null (아무것도 선택 안 됨)
│   ├── board[r][c] 있음 → selected = {r,c}, 효과음 'select'
│   └── board[r][c] 없음 → 무시
│
└── selected !== null (이미 선택됨)
    ├── 같은 셀 → selected = null (해제)
    ├── 비인접 셀
    │   ├── 유닛 있음 → 재선택
    │   └── 없음 → 선택 해제
    └── 인접 셀
        ├── 같은 티어 → doMerge()
        ├── 빈 셀 → doMove()
        └── 다른 티어 유닛 → 재선택
```

### 5.2 인접 판정
```javascript
function isAdjacent(r1, c1, r2, c2) {
  return (Math.abs(r1-r2) + Math.abs(c1-c2)) === 1;  // 상하좌우만 (대각선 X)
}
```

### 5.3 선택 시 하이라이트
- 합성 가능 셀: `.merge-target` (초록 테두리)
- 이동 가능 셀: `.move-target` (보라 테두리)

### 6.5 유닛 애니메이션

## 7. 오디오 시스템

### 7.1 구조
```javascript
const SFX = {
  ctx: null,      // AudioContext
  on: true,       // 효과음 ON/OFF
  vibOn: true,    // 진동 ON/OFF
  init()          // AudioContext 생성
  resume()        // suspended 상태 해제 (유저 인터랙션 필요)
  play(type)      // 효과음 재생
  vib(ms)         // 진동 (navigator.vibrate)
};
```

### 7.2 효과음 목록
| type | 용도 | 파형 | 주파수 | 특징 |
|------|------|------|--------|------|
| `click` | UI 클릭 | sine | 600Hz | 0.08초, 짧은 톤 |
| `select` | 셀 선택 | triangle | 520Hz | 0.12초 |
| `merge` | 일반 합성 | sine | 440→880Hz | 0.22초, 상승 글라이드 |
| `merge_high` | 고티어 합성 (5+) | sine | 660→1320Hz | 0.3초, 2음 하모닉스 |
| `combo` | 3콤보 이상 | sine | 550/770/990Hz | 3음 아르페지오 |
| `move` | 이동 | sine | 340Hz | 0.06초, 매우 짧음 |
| `gameover` | 게임 오버 | sawtooth | 300→80Hz | 0.6초, 하강 |
| `newbest` | 최고기록 | triangle | 440~960Hz | 5음 상승 아르페지오 |

### 7.3 진동 패턴
- 일반 합성: `15ms`
- 3콤보+: `[20, 15, 20]ms`
- 게임 오버: `[30, 30, 50]ms`

### 7.4 AudioContext 제한 대응
```javascript
// 브라우저 정책: 유저 인터랙션 없이 AudioContext 재생 불가
// 해결: 첫 pointerdown에서 resume() 호출
document.addEventListener('pointerdown', () => SFX.resume(), { once: true });
```

### 9.1 테마 시스템 (다크/라이트 모드)

```css
/* 테마 변수 정의 */

  --bg-body: #08081a;
  --bg-title: linear-gradient(...);
  --text-primary: #fff;
  --accent-purple: 147,112,219;   /* rgb 분리 → rgba()에서 활용 */

  /* ... 기타 변수 */

[data-theme="light"] {
  --bg-body: #eee8f5;
  --bg-title: linear-gradient(...);
  --text-primary: #1a1a2e;
  --accent-purple: 120,80,200;
  --accent-gold: 200,170,0;
  /* ... 기타 변수 */
}
```

## 14. 난이도 조절 가이드

게임 난이도를 조절할 때 아래 항목을 우선적으로 확인하세요.

### 14.1 스폰 관련 (난이도 핵심)

| 항목 | 현재값 | 위치 | 쉽게 ← → 어렵게 |
|------|--------|------|-----------------|
| 초기 배치 유닛 수 | 20개 | `INIT_UNIT_COUNT` | 줄이면 쉬움 ← → 늘리면 어려움 |
| 이동 시 스폰 수 | 3개 | `SPAWN_COUNT` | 줄이면 쉬움 ← → 늘리면 어려움 |
| 스폰 티어 다양화 | maxTier 기반 4단계 | `spawnUnits()` 내 확률 분기 | 종류 적으면 쉬움 ← → 많으면 어려움 |

**스폰 티어 확률 (현재값)**:
| 구간 | tier0 | tier1 | tier2 | tier3 | tier4 | tier5 |
|------|-------|-------|-------|-------|-------|-------|
| maxTier 0~2 | 70% | 30% | - | - | - | - |
| maxTier 3~4 | 50% | 30% | 20% | - | - | - |
| maxTier 5~6 | 40% | 30% | 20% | 10% | - | - |
| maxTier 7+ | 28% | 24% | 20% | 15% | 9% | 4% |

> 난이도를 올리려면: 높은 티어 비중 증가, 스폰 종류 추가
> 난이도를 내리려면: tier0 비중 증가, 스폰 종류 감소

### 14.2 아이템 획득 조건

| 아이템 | 현재 조건 | 위치 | 쉽게 ← → 어렵게 |
|--------|----------|------|-----------------|
| 최대 보유 수 | 3개 | `ITEM_MAX` | 늘리면 쉬움 ← → 줄이면 어려움 |
| 정화 ✨ | 왕관(tier 8) 합성 시 | `PURIFY_TIER` | 낮은 티어로 변경 ← → 높은 티어로 변경 |
| 폭탄 💣 | 매 7콤보 | `BOMB_COMBO_INTERVAL` | 숫자 줄이면 쉬움 ← → 늘리면 어려움 |
| 변환 🔄 | 매 100만점 | `MILESTONE_SCORE` | 점수 줄이면 쉬움 ← → 늘리면 어려움 |

### 14.3 명예의 전당

| 항목 | 현재값 | 위치 | 설명 |
|------|--------|------|------|
| 보너스 점수 | 1,000,000 | `HOF_BONUS` | 보상 크기 조절 |
| 발동 조건 | 최고 티어(tier 9) 합성 | `doMerge()` 내 `isHallOfFame` | 다른 티어로 변경 가능 |

### 14.4 빠른 난이도 조절 예시

**더 쉽게 만들려면**:
- `SPAWN_COUNT` 3 → 2, `INIT_UNIT_COUNT` 20 → 15
- `BOMB_COMBO_INTERVAL` 7 → 5, `PURIFY_TIER` 8 → 7(태양)

**더 어렵게 만들려면**:
- `SPAWN_COUNT` 3 → 4, `INIT_UNIT_COUNT` 20 → 24
- `BOMB_COMBO_INTERVAL` 7 → 10, `PURIFY_TIER` 8 → 9(삼지창)
- maxTier 7+ 스폰에 tier6까지 추가
